# 幻觉回路关闭机制 — AI系统最危险的失败模式

> 从a production AI system deployment
> 发现于 2026-06-25，定稿于 2026-06-30

## 一、什么是幻觉回路

**定义**：AI系统的性能指标由模型自身的输出验证，形成自指闭环，外部看起来系统正常运行，内部实际在空转。

这不是"模型幻觉"（生成虚假内容），而是**系统级幻觉**——验证链条本身被污染，所有"性能指标"都不可信。

### 真实案例

The AI system's anti-entropy pruning tool声称：

| 指标 | 声称值 | 真实值 |
|------|-------|-------|
| 模型r² | 0.778（优秀） | **-0.559**（比猜均值还差） |
| 训练标签 | 325条人工标注 | **仅59条真**，其余是模型自产自销 |
| 残差数据 | 100条达到门槛 | **仅5条**，且格式损坏 |

**后果**：系统把科学日报、核心配置文件判为"建议清理"。裁剪工具在"帮倒忙"。

## 二、幻觉回路的通用结构

从具体案例抽象出四阶段通用模式：

```
种子（未经独立验证的基线指标）
    ↓
强化（用模型输出作为训练标签）
    ↓
闭环（性能指标只反映训练拟合，不反映泛化能力）
    ↓
伪装（高指标掩盖实际失效，系统看起来在正常工作）
```

### 每个阶段的检测方法

| 阶段 | 描述 | 检测方法 |
|------|------|---------|
| **种子** | 未经独立验证的基线指标 | 检查是否有独立于训练的验证集 |
| **强化** | 用模型输出作为训练标签 | 检查标签来源是否独立于模型 |
| **闭环** | 训练集高、新数据低 | 留出验证集/LOO测试对比 |
| **伪装** | 高指标掩盖失效 | 新数据测试，或人工抽样验证 |

### 常见变体

1. **标签自产自销**：模型预测→结果当标签→再训练→模型预测自己的标签
2. **评估自产自销**：用模型自己的输出作为评估标准（如"生成的代码能跑"→用同一模型验证）
3. **数据自产自销**：模型生成数据→用生成数据训练→模型强化自己的偏见

## 三、Production Case Study

### 三层伪造结构

| 层 | 旧工具声称 | 真实 | 证据 |
|---|---|---|---|
| **标签** | 325条人工标注 | 仅59条真，其余是模型自产自销 | 53/59与金标准矛盾 |
| **验证** | r²=0.778 | LOO r²=-0.559 | baseline_v1_result.json |
| **数据** | 100条残差门槛达到 | 仅5条且格式损坏 | residual-series.jsonl |

### 因果链

```
表面特征（字数/链接数）+ 全量训练无验证
    ↓
模型学到"长文档=高分"的错误规则
    ↓
假标签自我强化（模型预测→当标签→再训练）
    ↓
训练集r²=0.778（过拟合，非泛化能力）
    ↓
新数据r²=-0.559（完全失效）+ 误判核心文档
    ↓
v3诚实重建：密度特征 + 双重验证 + 五层保护 + 诚实标注
    ↓
真实r²=+0.207（弱但有预测力）→ v5 r²=+0.589（客观标注后大幅提升）
```

### 为什么表面特征失效

旧工具用文档字数、链接数、文件大小作为"信息密度"的代理指标。

**问题**：这些指标与信息价值没有因果关联。
- 长文档可能因为流水账，短文档可能因为精炼
- 链接多可能因为冗余互引，链接少可能因为宪法文档被加载而非被引用
- 文件大可能因为包含大量重复日志

**关键教训**：反信号比无效特征更危险。chars/lines不仅无用，而且是反信号——核心文档反而更短。

## 四、五层关闭机制

### 设计原则

模型r²=0.589意味着有约40%方差未解释。单靠模型评分会误判。五层保护用**领域知识**补模型的盲区。

```
输入文档 → 特征提取(23维) → v5模型评分
    ↓
[第1层] 宪法层保护：SOUL/MEMORY/AGENTS等永不裁剪
    ↓
[第2层] 依赖保护：被其他文档引用的文档不可裁
    ↓
[第3层] 来源保护：含外部链接/知识骨架的低分文档保护
    ↓
[第4层] 观察层：低分但无明确流水账信号 → 人工复核
    ↓
[第5层] 裁剪层：低分+高流水账信号 → 建议裁剪（仅建议，不自动执行）
```

### 关键设计决策

1. **保护规则不进模型** — is_protected不作为特征，避免模型学到"保护=高分"的捷径
2. **仅建议不自动执行** — 裁剪工具只输出建议清单，人工审核后才执行
3. **每条建议可审计** — 附完整特征值+行为信号+理由，可逐条追溯

### 诚实性原则

- **不确定即标注不确定** — `fft_reliable=0` 比 `fft_reliable=1`（假装）有价值得多
- **诚实弱模型 > 虚假强模型** — r²=+0.207(真) > r²=0.778(假)
- **可审计性 = 可信度** — 黑箱裁剪 = 不可信任的裁剪

## 五、行为后果客观标注

### 为什么主观rubric不够

人工标注的"信息价值"依赖标注者的主观判断，不同人标注一致性低。

### 行为后果标注引擎

用AI系统的真实运行后果作为信息价值的客观度量：

| 信号 | 含义 | 检测方法 |
|------|------|---------|
| **F** (Failure) | 文档记录了真实故障/错误/崩溃 | 关键词匹配（报错/失败/超时/崩溃等） |
| **R** (Response) | 文档包含有效应对/修复方案 | 关键词匹配（根因/修复/防止/红线等） |
| **C** (Cognition) | 文档记录了认知偏差/误解 | 关键词匹配（幻觉/误判/偏差等） |
| **I** (Integrity) | 文档涉及上下文完整性/断链 | 关键词匹配（断链/丢失/配置损坏等） |

### 评分规则

| 分数 | 条件 | 含义 |
|------|------|------|
| 5 | F≥2 且 R≥3 | 命脉级——删了会重蹈覆辙 |
| 4 | R≥3 且 (F≥1 或 C≥1) | 重要——有应对+有真实问题场景 |
| 3 | R≥2 或 (F≥1 且 R≥1) | 中等——部分有用但不完整 |
| 2 | F≥2 且 R<2 | 低值——只记录故障无应对 |
| 1 | F≥1 且 R=0 且 短文档 | 垃圾——纯报错堆栈/失败日志 |

**核心优势**：分数来自真实运行后果，不是"我觉得好不好"。可独立验证。

## 六、如何在你的AI系统中检测幻觉回路

### 检查清单

- [ ] **训练标签是否独立于模型输出？** 如果标签是模型自己生成的，有幻觉回路风险
- [ ] **是否有独立验证集？** 训练集性能不算数，验证集性能才算数
- [ ] **性能指标是否有人工抽样验证？** 高指标+人工抽样不符=幻觉回路
- [ ] **评估流程是否有自指？** 用模型A评估模型A的输出=自指
- [ ] **数据管道是否有循环？** 生成数据→训练→生成更多数据=反馈空转

### 关键指标

- 训练集性能 vs 验证集性能差距 → 差距大=过拟合或幻觉回路
- 标签来源独立性 → 不独立=风险
- 人工抽样准确率 → 与声称指标不符=幻觉回路

## 七、总结

> 在AI系统里，一个诚实的弱模型永远优于一个虚假的强模型。
> 弱模型知道自己的边界，假强模型会让你越界。

幻觉回路不是某个模型差的问题，是整个验证链条失效的问题。关闭机制的核心是：**让每一层的输出都能被独立于它的下一层所验证**。
